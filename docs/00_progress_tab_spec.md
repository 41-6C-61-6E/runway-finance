# Spec: Progress Tab (Net Worth History Log)

> **Location**: Main Sidebar → Progress (third sidebar item)
> **Purpose**: Maintain a chronological, database-backed history log of the user's past assets, liabilities, and calculated net worth. It renders historical trend lines and handles progress point adjustments.

---

## 1. Visual Layout & Header Summary

The Progress tab layout matches the top-level chart structure of the Dashboard, but focuses on historical entries rather than future projections.

### 1.1 Header Metrics
Renders four textual metric widgets:
1.  **NET WORTH**: Large white bold text (e.g., `$1.24M`).
2.  **ALL TIME CHANGE**: Color-coded green/red text indicating absolute change since the oldest logged point (e.g., `▲ $785K (174.44%)`).
3.  **ASSETS**: Labeled gray header with total asset value (e.g., `$1.34M`).
4.  **LIABILITIES**: Labeled gray header with total liability value (e.g., `$101.5K`).

Located directly below the line chart:
*   **Section Header**: "Progress Points" with a vertical three-dots actions menu button.
*   **Sub-label**: "Created automatically when you update Current Finances."
*   **Log Table**: A tabular view of historical checkpoints:
    *   *Columns*:
        *   **Date**: Formatted as `MMM DD, YYYY` (e.g., `Mar 29, 2022`). Includes a sorting arrow (`Date ↙` or `Date ↓`) representing descending chronological order.
        *   **Net Worth**: Currency value (e.g., `$1.2M`).
        *   **Assets**: Currency value (e.g., `$1.3M`).
        *   **Liabilities**: Currency value (e.g., `$101.5K`).
        *   **Actions Button**: Vertical three-dots icon button (`v-btn icon`) at the end of each row.

## 2. Data Model & Schema

### 2.1 Progress Point Interface
```typescript
interface ProgressPoint {
  id: string;
  date: string; // ISO 8601 Date string (YYYY-MM-DD)
  netWorth: number;
  assets: number;
  liabilities: number;
  notes?: string;
  isAutomatic: boolean; // True if generated by system upon Current Finances baseline edits
}
```

### 2.2 Database Table DDL (PostgreSQL)
```sql
CREATE TABLE progress_points (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL,
    date DATE NOT NULL,
    net_worth DOUBLE PRECISION NOT NULL,
    assets DOUBLE PRECISION NOT NULL,
    liabilities DOUBLE PRECISION NOT NULL,
    notes TEXT,
    is_automatic BOOLEAN DEFAULT TRUE NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP NOT NULL
);

-- Indexing for fast chronological queries
CREATE INDEX idx_progress_points_user_date ON progress_points(user_id, date DESC);
```

## 3. Business Logic & Workflows

### 3.1 Automatic Point Creation
*   **Trigger**: Whenever the user edits any account balance in the **Today (Current Finances)** tab and saves changes.
*   **Action**: The system computes the aggregate assets, liabilities, and net worth today:
    $$\text{Assets}_{\text{total}} = \sum \text{SavingsBalance} + \sum \text{InvestmentBalance} + \sum \text{RealAssetValue}$$
    $$\text{Liabilities}_{\text{total}} = \sum \text{LoanBalance} + \sum \text{UnsecuredDebtBalance}$$
    $$\text{NetWorth}_{\text{total}} = \text{Assets}_{\text{total}} - \text{Liabilities}_{\text{total}}$$
*   **Upsert Logic**:
    *   If a progress point already exists for the *current calendar date* (today), update the assets, liabilities, and net worth columns of that row.
    *   If no point exists for today, insert a new row marked `is_automatic = true`.

### 3.2 Sorting and Formatting
*   **Default Sort**: Table rows are sorted by `date` in descending order (`Date ↓`).
*   **Row Options Action**: Clicking the three-dots button on a row opens a dropdown:
    *   `Edit Notes`: Exposes an input field to attach context (e.g. "Sold rental home", "Received yearly bonus").
    *   `Delete Entry`: Removes the database row. Removing a row forces the SVG chart to re-interpolate the line curve and re-calculates the "All-Time Change" summary metrics.
